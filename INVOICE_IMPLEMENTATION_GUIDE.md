# 📋 Guía Completa de Implementación de Facturación y Integración SRI

## 🎯 Resumen General

Este documento proporciona todo lo necesario para implementar un sistema de facturación completo compatible con el SRI (Servicio de Rentas Internas) de Ecuador.

## 📊 ARQUITECTURA DE DATOS

### Tablas Requeridas en Supabase

```sql
-- 1. FACTURAS (Invoices)
CREATE TABLE invoices (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id UUID NOT NULL REFERENCES companies(id),
  user_id UUID NOT NULL REFERENCES users(id),
  invoice_number VARCHAR(50) UNIQUE NOT NULL,
  sequential INT NOT NULL,
  establishment VARCHAR(3),
  point_of_sale VARCHAR(3),
  issue_date TIMESTAMP NOT NULL,
  subtotal_amount DECIMAL(10,2),
  discount_amount DECIMAL(10,2),
  taxable_amount DECIMAL(10,2),
  tax_amount DECIMAL(10,2),
  total_amount DECIMAL(10,2),
  tax_rate DECIMAL(5,2),
  payment_method VARCHAR(20),
  customer_id UUID REFERENCES customers(id),
  customer_name VARCHAR(255),
  customer_email VARCHAR(255),
  customer_phone VARCHAR(20),
  status VARCHAR(50) DEFAULT 'pending', -- pending, approved, sent, rejected, cancelled
  sri_status VARCHAR(50) DEFAULT 'pending', -- pending, sent, accepted, rejected, error
  transaction_id VARCHAR(255),
  access_key VARCHAR(50), -- Generated by SRI
  xml_receipt TEXT, -- XML compliant with SRI
  reference VARCHAR(255),
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 2. DETALLES DE FACTURA (Invoice Details)
CREATE TABLE invoice_details (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  invoice_id UUID NOT NULL REFERENCES invoices(id) ON DELETE CASCADE,
  product_id UUID REFERENCES products(id),
  product_code VARCHAR(50),
  product_name VARCHAR(255) NOT NULL,
  quantity INT NOT NULL,
  unit_price DECIMAL(10,2),
  discount_percent DECIMAL(5,2),
  discount_amount DECIMAL(10,2),
  subtotal DECIMAL(10,2),
  tax_rate DECIMAL(5,2),
  tax_amount DECIMAL(10,2),
  total DECIMAL(10,2),
  created_at TIMESTAMP DEFAULT NOW()
);

-- 3. ÍNDICES PARA PERFORMANCE
CREATE INDEX idx_invoices_company ON invoices(company_id);
CREATE INDEX idx_invoices_date ON invoices(issue_date);
CREATE INDEX idx_invoices_sequential ON invoices(company_id, sequential);
CREATE INDEX idx_invoice_details_invoice ON invoice_details(invoice_id);
```

## 🔌 INTEGRACIÓN EN POSInterface.jsx

### Cuando se Realiza una Venta

```javascript
const completeSale = async () => {
  try {
    // 1. Obtener configuración de facturación
    const billingConfig = await getBillingConfig(currentUser.company_id);

    // 2. Validar datos
    const validation = validateInvoiceData(
      { total_amount: total },
      billingConfig
    );
    if (!validation.isValid) {
      showToast('error', validation.errors.join(', '));
      return;
    }

    // 3. Obtener siguiente secuencial
    const sequential = await getNextInvoiceSequential(currentUser.company_id);

    // 4. Crear factura
    const invoice = await createInvoice({
      company_id: currentUser.company_id,
      user_id: currentUser.id,
      invoice_number: generateInvoiceNumber(billingConfig, sequential),
      sequential,
      establishment: billingConfig.establishment,
      point_of_sale: billingConfig.pointOfSale,
      subtotal_amount: subtotal,
      discount_amount: discount,
      taxable_amount: taxableAmount,
      tax_amount: tax,
      total_amount: total,
      tax_rate: taxRate,
      payment_method: paymentMethod,
      customer_id: customerData.id || null,
      customer_name: customerData.name,
      customer_email: customerData.email,
      customer_phone: customerData.phone,
      transaction_id: `TX-${Date.now()}`,
      reference: ''
    });

    // 5. Crear detalles de factura
    for (const item of cart) {
      const itemDiscount = item.sale_price * item.quantity * (discountPercent / 100);
      const itemSubtotal = item.sale_price * item.quantity - itemDiscount;
      const itemTax = itemSubtotal * (taxRate / 100);

      await createInvoiceDetail({
        invoice_id: invoice.id,
        product_id: item.id,
        product_code: item.code,
        product_name: item.name,
        quantity: item.quantity,
        unit_price: item.sale_price,
        discount_percent: discountPercent,
        discount_amount: itemDiscount,
        subtotal: itemSubtotal,
        tax_rate: taxRate,
        tax_amount: itemTax,
        total: itemSubtotal + itemTax
      });
    }

    // 6. Generar XML para SRI
    const details = await fetchInvoiceDetails(invoice.id);
    const company = await fetchCompanyById(currentUser.company_id);
    const xmlData = generateSRIXML(invoice, details, company);

    // 7. Actualizar factura con XML (opcional, enviar a SRI)
    await updateInvoiceStatus(invoice.id, 'approved', 'pending');

    showToast('success', `Factura generada: ${invoice.invoice_number}`);

    // 8. Limpiar carrito
    setCart([]);
    setPaymentMethod('cash');
    setDiscountPercent(0);
    setShowPayment(false);
  } catch (error) {
    showToast('error', error.message);
  }
};
```

## 📱 FLUJO DE COMPRA A FACTURA

```
1. USUARIO REALIZA COMPRA EN POS
   ↓
2. CLIC EN "PROCESAR PAGO"
   ↓
3. SISTEMA VALIDA CONFIGURACIÓN DE FACTURACIÓN
   ↓
4. OBTIENE SIGUIENTE SECUENCIAL
   ↓
5. CREA REGISTRO EN TABLA "invoices"
   ↓
6. CREA DETALLES EN TABLA "invoice_details"
   ↓
7. GENERA NÚMERO DE FACTURA (RUC-ESTAB-POS-SECUENCIAL)
   ↓
8. GENERA CLAVE DE ACCESO (24 dígitos)
   ↓
9. GENERA XML COMPLIANT SRI
   ↓
10. ALMACENA XML EN BD
   ↓
11. ACTUALIZA ESTADO: "approved" / "pending"
   ↓
12. MUESTRA COMPROBANTE AL USUARIO
   ↓
13. LISTA PARA ENVIAR AL SRI (PASO FUTURO)
```

## 🔐 SEGURIDAD Y VALIDACIÓN

### RUC Validation

```javascript
// El RUC debe tener exactamente 13 dígitos
// Los primeros 10 dígitos son el número de cédula/RUC
// El dígito 11 es el dígito verificador
// Los dígitos 12-13 son "001"

// Ejemplo válido: 1723456789001
// Donde:
// 1723456789 = Cédula base
// 0 = Dígito verificador
// 01 = Establecimiento/sucursal

validateRUC("1723456789001") // true
```

### Clave de Acceso

```javascript
// Formato: AAAAMMDDRUC(13)-TIPO(2)-SECUENCIAL(9)-VERIFICADOR(1)
// Total: 24 dígitos
// Ejemplo: 240715172345678901010000000011

// GENERADA AUTOMÁTICAMENTE POR:
generateAccessKey({
  issue_date: "2024-07-15T...",
  ruc: "1723456789001",
  sequential: 1
})
```

## 🔄 SIGUIENTE: ENVÍO AL SRI

### Pasos para Implementar (Futuro)

1. **Integración con WS-SRI**
   - Endpoint: `https://cel.sri.gob.ec/comprobantes-electronicos-ws/AutorizacionComprobante...`
   - Método: SOAP/REST
   - Requiere: Usuario/Contraseña del SRI

2. **Proceso de Autorización**
   - Enviar XML firmado
   - Esperar respuesta del SRI
   - Actualizar estado: "sent", "accepted", o "rejected"

3. **Recibir Comprobante Autorizado**
   - SRI devuelve XML autorizado
   - Guardar en BD
   - Generar PDF desde XML

4. **Disponibilizar al Cliente**
   - Enviar PDF por email
   - Mostrar en portal de cliente
   - Generar código QR

## 📋 ESTADO DE FACTURA

```javascript
Estados Disponibles:
- pending:    En edición
- approved:   Aprobada, lista para enviar
- sent:       Enviada al SRI
- accepted:   Aceptada por SRI ✓
- rejected:   Rechazada por SRI ✗
- cancelled:  Cancelada

Estados SRI:
- pending:    No enviada aún
- sent:       En proceso de validación
- accepted:   Aceptada ✓
- rejected:   Rechazada ✗
- error:      Error de comunicación
```
